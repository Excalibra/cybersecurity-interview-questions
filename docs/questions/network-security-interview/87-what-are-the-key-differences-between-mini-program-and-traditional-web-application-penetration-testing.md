# Q: What are the key differences between mini-program and traditional web application penetration testing?

**A:** The key differences are:

1.  **Focus Area:** Mini-program testing focuses heavily on the application layer, including API security, authentication, and permission controls. Traditional web testing can encompass a broader range, including network-level infrastructure.
2.  **Objective:** The goal of mini-program testing is to find security flaws within the mini-program itself. Traditional web testing aims to find vulnerabilities in the underlying system, web application, and its environment.
3.  **Techniques:** Mini-program testing is more application-layer focused, while traditional web testing often involves network scanning, infrastructure analysis, and database security assessment.
